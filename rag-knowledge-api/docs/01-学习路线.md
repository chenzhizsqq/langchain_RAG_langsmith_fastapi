# 学习路线

如果你是第一次系统接触这一套，建议按下面顺序走。

## 第 1 步：先分清五个关键词的职责

- `LangChain`：流程编排层
- `RAG`：应用模式
- `LangSmith`：观测和调试
- `FastAPI`：后端接口层
- `Vector DB`：语义检索存储层

很多人一开始学混乱，就是因为把这些词都当成“同一种东西”。它们不是同一层。

## 第 2 步：先跑通，再深挖

建议你先做这几件事：

1. 启动 FastAPI 服务
2. 导入内置 seed docs
3. 提一个问题
4. 看返回的 answer 和 sources
5. 再去 LangSmith 看 trace

先有整体感，再回来看源码，你会更容易理解每个文件在干什么。

## 第 3 步：按链路读代码

建议从这里开始看：

1. `app/main.py`
2. `app/loaders.py`
3. `app/rag_service.py`
4. `data/seed_docs/*.md`

因为真正的运行路径就是：

FastAPI 接口 -> 文档加载 -> 文档切块 -> 向量化 -> 向量检索 -> 生成答案

## 第 4 步：学会判断问题出在哪一层

当答案不理想时，不要只说“模型不行”。应该依次检查：

1. 文档有没有成功导入
2. 切块是否过大或过碎
3. 检索出来的 sources 是否相关
4. prompt 有没有明确要求基于 context 回答
5. top_k 是否太小

## 第 5 步：再做扩展

当这套最小项目你已经能解释清楚之后，再继续做：

- 对话历史
- metadata 过滤
- 重排序
- 权限控制
- 前端页面
- 评估集和自动化测试

先把基础架起来，比堆更多名词有用得多。
