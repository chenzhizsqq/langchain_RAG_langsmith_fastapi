# LangSmith 是什么

LangSmith 是 LangChain 生态里的可观测性和调试平台。很多初学者第一次做 RAG 时，只看最终答案对不对，但这其实不够。你真正需要知道的是：检索到了哪些片段、prompt 是怎么拼出来的、模型花了多久、输出有没有偏题。

LangSmith 的价值就在这里。它把每一次调用拆开，让你看到每一个步骤。对 RAG 项目来说，这尤其重要，因为问题经常不在模型，而在检索阶段。例如，文档切块太大、top_k 太小、向量库里数据不干净，都可能导致最后答案变差。

在这个教程里，只要设置 `LANGSMITH_TRACING=true` 和 `LANGSMITH_API_KEY`，你就能在 LangSmith 界面看到链路。你会看到文档导入、检索、模型调用这些步骤。这样你就不再只是“感觉有问题”，而是能明确知道问题发生在哪一步。

如果把 LangChain 理解成 AI 应用的流程层，那么 LangSmith 就是这个流程层的观察和调试工具。
