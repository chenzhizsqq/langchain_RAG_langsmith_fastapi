# RAG 是什么

RAG 的全称是 Retrieval-Augmented Generation，意思是“检索增强生成”。它的核心思想很简单：不要只靠模型记忆，而是先从你自己的资料里找到相关内容，再让模型基于这些内容回答。

一个基础 RAG 流程通常分成两段。第一段是索引阶段：读取资料、切块、生成 embedding、写入向量数据库。第二段是问答阶段：收到用户问题、把问题做成查询、去向量库检索相似片段、把片段和问题一起交给大模型生成答案。

RAG 能解决两个常见问题。第一，模型的通用知识不一定包含你的私有资料。第二，纯模型问答容易出现幻觉，RAG 可以让答案更贴近给定材料。

但是 RAG 不是魔法。RAG 的效果高度依赖切块是否合理、embedding 模型是否合适、向量检索是否找到了真正相关的内容，以及 prompt 有没有明确要求模型基于上下文回答。如果检索错了，后面的生成通常也会错。
