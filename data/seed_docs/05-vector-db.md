# Vector DB 是什么

向量数据库的作用，是把文本对应的 embedding 向量存起来，并在查询时根据向量相似度找出语义最接近的内容。它和传统数据库不冲突，只是解决的问题不同。传统数据库擅长精确查询，向量数据库擅长语义近邻搜索。

在 RAG 场景里，Vector DB 常见工作包括：存储文本切块及其 metadata、根据用户问题做相似度搜索、返回最相关的若干片段。很多产品还支持 metadata 过滤、混合检索、重排序等能力。

这个项目选择 Chroma，原因是它对初学者非常友好。你可以本地直接跑，先把“embedding 到检索”的闭环打通，不需要先购买云服务，也不需要先管理更多部署细节。

理解 Vector DB 的关键，不是记住某个产品名字，而是明白它在 RAG 中负责“把问题和资料做语义匹配”。只要这一层理解清楚了，以后换成 Pinecone、Qdrant、Weaviate 或 pgvector，整体思路都不会变。
