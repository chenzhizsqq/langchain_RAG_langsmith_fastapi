# FastAPI 在 AI 项目里的角色

FastAPI 是一个现代 Python Web 框架，很适合把 AI 能力做成后端接口。你可以把模型调用、RAG 检索、文件上传这些能力都包成 HTTP API，然后前端、移动端或者别的服务都来调用。

对初学者来说，FastAPI 的优势有三点。第一，接口定义清晰，参数和返回值都可以用 Pydantic 描述。第二，自带 Swagger 文档，启动后访问 `/docs` 就能在线测试。第三，它和 Python 生态非常贴近，集成 LangChain、向量库、文件处理都很自然。

在这个项目里，FastAPI 负责暴露几个最关键的入口：健康检查、导入内置资料、导入文本、上传文件、执行问答、查看向量库统计信息。你可以把它理解为这套 RAG 服务的门面层。

以后如果你想接网页前端，最简单的做法就是继续保留 FastAPI 作为后端，把 React、Vue、iOS 或 Android 作为调用方即可。
